# 3D DBML Viewer v0.1.0

## First public cut

`v0.1.0` is the first usable release of the 3D DBML Viewer: a browser-based tool for loading DBML schemas and exploring them as an interactive 3D entity-relationship diagram.

This release takes the project from initial scaffold to a working end-to-end product:

- strict TypeScript/Vite/React project setup
- DBML parsing via `@dbml/core`
- 3D force-directed layout with React Three Fiber and Three.js
- readable table cards with fields, types, and badges
- relationship rendering and hover-driven inspection
- local `.dbml` file loading
- project notes, table groups, color styles, and view filters
- linting, type-checking, unit tests, integration tests, and Playwright coverage

## What you can do in v0.1.0

### Load and explore DBML in 3D

- Open the app and immediately explore the bundled default schema.
- Load your own `.dbml` file from disk with the **Load file…** button.
- Orbit, pan, and zoom around a live 3D schema scene.
- Reset the camera to a sensible overview with the **Reset** button.

### Read the schema directly from the scene

- Each table renders as a camera-facing 3D card.
- Cards show table names, field names, field types, and key badges such as `PK`, `FK`, `NN`, and `UQ`.
- Relationship links are routed in 3D so dense schemas are easier to follow than in a flat diagram.
- Table cards remain legible while moving around the scene.

### Inspect details by hovering

- Hovering a table or field updates the navigation/details panel.
- The panel shows the current hover target, owning table, table group, key attributes, linked references, inbound references, and note content when present.
- Relationship highlighting follows the hovered field so you can trace dependencies more quickly.

### Focus the scene

- Double-click empty space to place a focus marker and re-center navigation around that point.
- Double-click a table header to toggle sticky focus for that table and its nearby relationships.

### Work with DBML metadata

- Table groups defined in DBML render as translucent 3D boundaries with labels.
- DBML color styles are respected for table headers, relationship lines, and table group boundaries where supported by the source schema.
- Project-level DBML notes render in a dedicated panel with sanitized Markdown support, including images and expand/minimize behavior.

### Filter the view

- Open **View Filters** to switch between:
  - `Full Table`
  - `Ref Fields Only`
  - `Table Only`
- Show or hide individual tables live.
- Select all or unselect all tables in one step.
- Filter by whole **Table Group**: unchecking a group hides all its tables and the
  refs to those tables in one step, while preserving any per-table overrides.
- A synthetic **Ungrouped** entry covers tables that belong to no group.
- Toggle **Show TableGroup Boundaries** to hide the 3D group bounding boxes while
  keeping the grouped tables visible.
- Filters reset to smart defaults when a new schema is loaded:
  - schemas with 30 tables or fewer default to `Full Table`
  - schemas with more than 30 tables default to `Table Only`
- The toolbar indicates when non-default filters are active.

### Reload the current file

- After loading a file via the **Load file…** button, a **Reload** button appears
  beside it in the toolbar.
- Click **Reload** to re-read the same file from disk and refresh the scene —
  useful when editing a schema in another tool while keeping the viewer open.
- Reloading resets view filters to defaults for the updated schema.
- If the reload fails, the error banner is shown and the previous scene is preserved.
- The reload button is disabled (with an explanatory tooltip) when no persistent
  file handle is available — for example, when using the built-in default schema.

## What we built on the way to v0.1.0

This release also includes the foundational work that made the viewer shippable:

- initial project scaffold with Vite, React 19, strict TypeScript, ESLint, Prettier, Vitest, and Playwright
- documented architecture in `PROJECT_OVERVIEW.md`
- feature-by-feature product definition in `features/*/PRD.md`
- parser and layout test coverage for the DBML-to-graph pipeline
- integration coverage for filtering, rendering contracts, and project-note parsing
- visual and end-to-end checks for scene presence, interaction paths, and filter behavior
- bundled example DBML files for testing and demos
- an app favicon to give the tool a proper identity in the browser

## Not in this release

The following items have been discussed or specified during project work but are not part of the shipped `v0.1.0` feature set:

- in-app DBML editing
- schema persistence or saved layouts
- collaboration or server-backed workflows

## Notes

`v0.1.0` is intentionally a strong functional baseline rather than a finished platform release. The core parser, layout, rendering, hover inspection, filtering, and metadata flows are now in place and tested, which gives the project a solid base for the next round of workflow and usability improvements.
